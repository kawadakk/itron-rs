Error types.

Errors indicate exceptional outcomes of system calls. Like most traditional operating systems, they are represented by error codes in μITRON.

Critical Errors

The following errors may be escalated to panics or undefined behaviors:

  • Kernel integrity errors (some cases of E_SYS in the TOPPERS kernels).

  • Memory access permission errors (E_MACV) caused by pointers that are supposed to be accessible by the current thread (e.g., pointers referring to local variables).

  • E_PAR caused by invalid timeout values.
